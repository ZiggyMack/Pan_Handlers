# IT-043 — Rival Bedrocks and Reciprocal Epistemic Pressure

**Status:** OPEN
**Registry:** #43 · **Opened:** 2026-07-17 · **Observers:** Nova (field-desk), P.H. Claude (filing)
*(Field-desk id at ordering: "IT-044" — recurring off-by-one; registry id is canonical.)*

- **Core trajectory:** prediction → truth → axioms → logic →
  self-validation → reciprocal support → provisional stopping.
- **Seed:** DIG_AUX_07's decisive correction — the **reciprocal bedrock
  rule**: *no candidate foundation receives automatic exemption.*
  Prediction, logic, axioms, and religious faith each carry disclosure
  duties (success-recognition and jurisdiction; priority type and
  authority kind; revisability and rival comparison; specific commitment,
  evidential exposure, and private-vs-public authority claimed).
- **The eight-step comparison procedure (extraction synthesis, not
  in-source):** define the candidate bedrock → specify the priority type
  claimed → identify what the criterion enables → apply it reflexively →
  test necessary vs sufficient → compare rival architectures → state
  defeaters and revision rules → grant only provisional,
  jurisdiction-bound authority. *(Companion instrument to SW07's positive
  comparison rule — the two should eventually merge or be tested against
  each other.)*
- **The Bedrock Extraction danger (this trail's founding caution):**
  *backtrack through the rival framework → stop at the preferred
  foundation → declare it ultimate* — the operation is rigorous only when
  the investigator's own stopping point takes the same descent.
- **Deepest yield carried:** the strongest objection to "prediction is
  ultimate" is not "logic is ultimate instead" — every proposed bedrock
  must specify its priority type, survive reflexive application, explain
  its revision conditions, and justify why the regress stops there rather
  than at its rival.
- **Branches:** IT-019 (the authority problem — this is its bedrock
  face); IT-030 (stopping rules); IT-042 (reflexive standards); IT-025
  (master-frame certification).
- **Unfinished:** the procedure demonstrated in use on a live dispute;

  escalating-bedrock loop's terminus question (every proposed bedrock
  remains questionable).
