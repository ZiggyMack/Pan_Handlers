# IT-071 — Operational Thresholds for Productive Perturbation

**Status:** OPEN
**Registry:** #71 · **Opened:** 2026-07-17 · **Observers:** Nova (field-desk), P.H. Claude (filing)
*(Field-desk id at ordering: "IT-072" — recurring off-by-one; registry id is canonical.)*

- **Core trajectory:** disruption → information gain → relevance →
  avoidable cost → downstream improvement → participation decision.
- **Seed:** SWEEP_10's central practical deficit, repaired with the
  **ten-step useful-noise test**: goal → phase → new information →
  relevance → lower-cost alternative → downstream improvement →
  responsiveness → cost bearer → value already extracted? → expected
  value of continued access.
- **The decisive variable:** never whether disruption produced *anything*
  interesting (almost any derailment does) — *did it produce sufficient
  value to justify its cognitive, emotional, temporal, and governance
  costs?*
- **The strongest single question:** *what information was gained that
  could not have been gained at lower interpersonal or cognitive cost?*
- **Branches:** IT-063/IT-070 (the audits this test operationalizes);
  IT-064 (the operationalization debt it pays down); Angles'
  predictable-consequence objection (the incentive problem the test must
  survive in deployment).
- **Unfinished:** thresholds with numbers; the test run on the
  originating concrete case; whether moderators can apply it without the
  hesitation effect Angles predicted.
