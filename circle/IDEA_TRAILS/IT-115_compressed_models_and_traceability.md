# IT-115 — Compressed Models and Traceability

**Status:** OPEN (Idea Trail)
**Registry:** #115 · **Opened:** 2026-07-17 · **Observers:** Nova (field-desk), P.H. Claude (filing)

- **Core datum (Z-DIRECT):** Ziggy retains structural gist, loses
  wording and sequence, and reconstructs later from a compressed
  model — a demonstrated cognitive style with real strengths
  (abstraction, pattern recognition, generalization) and named risks
  (source-monitoring errors, reconstruction drift, omitted
  qualifiers, motive projection, **later assistant language entering
  the remembered model**).
- **The two-sided rule (binding):** *loss of transcript detail ≠ loss
  of understanding; structural memory ≠ transcript-level accuracy;
  recognizable pattern ≠ verified motive* — and the closure caution:
  Structural Compression is profile-ready **as a demonstrated
  operation, never as evidence the compressed model is accurate**.
  Compressed reconstruction must not become self-authenticating.
- **The seven-practice traceability protocol (completion
  architecture, A1):** separate quotation from paraphrase; label
  direct / reported / interpreted; preserve anchors; record
  wording-confidence; gist ≠ historical sequence; recheck against
  source; state what would force model revision.
- **The relationship face:** the Compression–Precision loop — *gist
  retained → details fade → model asserted → opponent disputes
  details → the detail dispute read as pattern-avoidance → the model
  hardens* — traceability decreasing while confidence increases.
  "You think in models; he argues in transcripts" stays **Nova's
  compression**, never an established Ziggy–Grant contrast.
- **Kinship:** IT-112 (retained justification states — the
  first-person face of the same territory); the Grant-model
  disconfirmation debt (SW14/SW15); the co-constructed-label
  authority loop (SW15).
- **Unfinished:** whether screenshot preservation actually corrects
  drift; a revision rule for when transcript and model conflict.
