# IT-038 — Formal Authority Leakage in Technical Metaphor

**Status:** OPEN (methodological — a Failure-Atlas-adjacent trail)
**Registry:** #38 · **Opened:** 2026-07-17 · **Observers:** Nova (field-desk), P.H. Claude (filing)
*(Field-desk id at ordering: "IT-039" — recurring off-by-one; registry id is canonical.)*

- **Core trajectory:** technical term → metaphorical extension → borrowed
  mathematical authority → delayed correction → **residual conceptual
  influence**.
- **Seed:** COMPOUND_SWEEP_07's named source-quality hazard,
  **formal-authority residue**: Nova validated "high entropy meaning"
  before correcting it, and the mathematical vocabulary had already shaped
  several downstream claims by the time the formal meaning was withdrawn.
- **The mechanism:** a technical metaphor lends rhetorical precision; the
  formal grounding is later abandoned; the metaphor keeps some of the
  borrowed authority — *the correction arrives; the residue stays.*
- **Why it matters beyond this dig:** this is the excavation's clearest
  instance of the validation→overreach→correction relationship loop
  producing *persistent* conceptual contamination — the source-quality
  failure mode that the accommodation-correction loop (SWEEP_04) only
  partially captured.
- **Registered instances:** entropy/"semantic entropy" (SW07, corrected
  in-thread, residue audited at Stages 3–4); candidate kin: "mechanism"
  language in AUX_04 (permanently downgraded); Bell technical-risk gate
  (IT-016). Same-extractor caveat: all instances so far pass through Nova.
- **Detection heuristic (proposed):** when a formal term appears outside
  its formal jurisdiction, ask — *what quantity, theorem, or measurement is
  actually being invoked? if none, what work is the term's prestige doing?*
