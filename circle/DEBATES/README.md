## Record Format

Filename: `DB_YYYYMMDD_<slug>.md` (date = when the debate happened, best known).

```markdown
# Debate: <topic>
**Date:** when it happened · **Recorded:** when this file was written (flag the gap)
**Medium:** voice / chat / thread / in-person
**Participants:** members present
**Recorder:** who wrote this record (observer attribution)
**Fidelity:** TRANSCRIPT / DETAILED NOTES / RECONSTRUCTION FROM MEMORY

## What was argued
Position-level summary per participant (link POSITION_LEDGER entries).

## Moves observed
Operator-level notes: who deployed what (CO-### refs), in what order.
Sequences logged here feed OPERATOR_MAP §E — remember they stay hypotheses
until seen across multiple debates.

## Outcome state
Resolved / revised / contested-and-standing / abandoned. Contested ≠ defeated.

## Loose threads
What was left hanging — feeds IDEA_TRAILS and OPEN_QUESTIONS.
```

## Fidelity Discipline

A `RECONSTRUCTION FROM MEMORY` record caps every claim derived from it at
`INFERRED`. Only `TRANSCRIPT`-fidelity records can support `DIRECT`.
